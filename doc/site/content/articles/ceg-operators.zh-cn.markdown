## 基础

您可以通过指定一系列操作符，而不仅仅是一个数字，来自定义某些CEG条目。
这使您能够应用有限的逻辑来定制爆炸效果。
每个操作由一个字符串表示。

### 原始数据

一个数字就表示那个数字。例如：
```lua
sizeGrowth = "3",
```
这将 `sizeGrowth` 设置为3。到目前为止，很简单。

某些参数，尤其是三维向量，可以接受多个值。在这种情况下，我们用 `,` 逗号操作符分隔它们。
例如：
```lua
pos = "1, 2, 3",
```
这将位置向量设置为 x=1，y=2，z=3。

### 运行值

有一个隐式的运行值，初始为0，所有操作符都基于此进行运算。
原始数字实际上是一种“无操作符”的操作符，它对当前值执行简单的加法运算。
因此，人们可能会认为上述例子实际上 `0 +3` 是 `sizeGrowth` 和 `0 +1`, `0 +2`, 和 `0 +3` 作为组件的 `pos` 分别对应向量。


为了说明这一点的重要性，考虑以下例子。请注意，这里没有任何 `,` 在这些之间！
```lua
sizeGrowth = "1 2 3",
```
结果是6。这是因为每一个都是加法： `0 +1 +2 +3`, 净值为6。

类似的:
```lua
pos = "1 2, 3 4, 5 6",
```
这将组件分别设置为3（0 +1 +2）、7（0 +3 +4）和11（0 +5 +6）。


把一堆原始数字并列在一起并不太有意义，因为我们本可以直接写出它们的和，但当混合运算符时，这个原则就开始变得重要了。

### 随机（`r`）

该 `r` 运算符也对运行值进行操作，但会添加一个介于0到操作数之间的随机值。
举个例子， `r4` 返回 0 到 4 之间的随机值。实用提示：
 * 非常有助于使爆炸效果看起来不那么人工。
 * 如果你不想从0开始滚动，只需添加偏移量即可（记住，原始的无操作符数字会执行加法运算）。
所以， `3 r4` 给出一个介于3到7之间的值。
 * 一个常见的需求是让负值滚动，例如使定向粒子可以朝任意方向移动。
在这种情况下，也应使用偏移量。一个常见的习惯是例如使用 `-15 r30` ±15 转动。
 * 数值是均匀分布的，但如果你具备一定的统计知识，可以通过叠加骰子来对其进行调整。
例如， `r6` 会产生一个平坦且均匀的 0 到 6 的分布 `r3 r3` 会形成一种类似三角形的分布，其中 3 的出现概率远高于 0 或 6 而 `r2 r2 r2` 则更加平滑，更接近钟形曲线。  
实际上，这种方法似乎使用得非常少，通过基本方法无法使分布变得不对称，但可以通过下面更高级的方法实现。

### 索引（`i`）

`i` 操作符将它的操作数乘以粒子的索引，并将结果加到当前值上。
当生成器创建多个相同类型的粒子时，每个粒子都会被分配一个递增的索引：0、1、2，依此类推。

例如，如果一个爆炸生成了4个粒子，且 `size = "3 i2"`，那么这些粒子的大小分别为3、5、7和9。

 * 适用于在类似线状或圆锥形的结构中生成物体。
 * 通过粒子数量来缩放爆炸效果非常有用，因为低阶粒子的行为与之前相同。
 * 记住，这不会乘以运行值，或任何其他内容，仅是操作数。

### 伤害（`d`）

`d` 操作符会将其操作数乘以爆炸的“伤害值”。
例如，`d0.1` 对于造成 100 点伤害的爆炸将净得 10 点，而对于造成 500 点伤害的爆炸则为 50 点。

一些实用建议：
 * 对于普通武器，这是“默认”伤害值。请注意，如果将其视为“特性”护甲等级（因为它们无法拥有真正的护甲等级）时需特别小心！
 * 对于CEG导弹轨迹，损坏即为弹丸剩余的TTL。因此，你可以让导弹轨迹燃烧殆尽，例如这样实现效果。
 * 确保伤害调整（无论是游戏内如增益或升级，还是游戏外如补丁后平衡性调整）都能在视觉上微妙地体现出来。
 * 这样你就可以为相似类型的多种武器重复使用同一个CEG，从而实现出色的视觉一致性。
 * 现有游戏通常希望为每种相似武器设置独立的效果，因此这种操作方式在示例中相当少见。

## 高级

除了运行值外，您还可以访问一个包含16个“槽位”的缓冲区，用于保存数值以供后续使用。
除了支持复杂的数学运算外，这些功能还允许您将同一个数值重复用于向量的多个组件（跨越逗号分隔符边界时，通常会重置运行值）。
此外，还有一些运算符涉及比简单加法更复杂的数值操作。

### Yank (`y`), add (`a`), and multiply (`x`)

`y` 操作符将当前运行值保存到指定索引处的缓冲区中，并将运行值重置为 0。
`a` 操作符将给定缓冲区中的当前运行值相加。
`x` 运算符将运行值乘以给定缓冲区的值。

示例：
 * `10 r20 y7  5 r10 x7`. 先随机生成一个10到30之间的值（参见前面的课程），将其保存到缓冲区#7中（该缓冲区会将当前值重置为0），然后生成另一个5到15之间的随机数，再将其乘以缓冲区#7中的内容值（即上一次掷出的值）。通常， `foo yN bar xN` 这种模式用于对  `foo` 和 `bar` 进行相乘运算。
 * `r10 y9 a9, 0, a9`. 随机生成一个值，将其保存到缓冲区 #9，由于重置操作，该值会被重新加载。然后将该值用于向量的第三个分量。通过这种方式可以得到对角向量。

### 正弦（`s`）

`s` 操作符将其操作数视为振幅，当前运行值视为相位，并将当前运行值替换为计算结果。  
例如 `3 s2` 约等于 0.28，因为这是 `2 * sin(3 弧度)`.

 * 只有在 `r`、`i` 或 `d` 这样的不可预测性来源时，才真正有意义。
 * 没有单独的余弦运算符，但你可以通过 `cos(x) = sin(π/2 + x)` 来实现一个“ ghetto 余弦”，即只需输入 `1.57 sX` 而不是仅仅 `sX`。
 * 适合用于创建圆形或球形的体积效果（对于非体积效果，有诸如 `emitRot` 等基本的扩散参数）。

### 锯齿/模数（`m`）

对运行值应用取模运算符。例如：
 * `1 m7` 是1
 * `6 m7` 是6
 * `7 m7` 是0
 * `8 m7` 是1
 * `13 m7` 是6
 * `14 m7` 是0

结合 `i` 操作符，或 CEG 路径中的 `d` 操作符，可以实现周期性或“线条虚线”风格的效果，因为这些情况下会连续生成多个粒子，并让锯齿波形以连续的数值进行处理。  
例如，`numParticles = "d0.125 m1 0.125"` 可在 CEG 路径中每 8 帧生成一个粒子。

### 离散化（`k`）

将运行值截断为操作数的倍数：
 * `1 k7` 是0
 * `6 k7` 是0
 * `7 k7` 是7
 * `8 k7` 是7
 * `13 k7` 是7
 * `14 k7` 是14

你可以用它来进行比较，并实现类似底层的布尔逻辑，只要能为运行值提供某种上限即可。  
这确实是一种相当愚蠢的做法，如果你已经到了这种程度，不妨考虑一下，不如直接通过 Lua 创建粒子会更好。
* 假设你想检查“如果 x >= 123”，并且可以认为 x < 10000。
* 从 `(...) 9877 k10000 -9999`
* 现在运行值为0或1，取决于它在之前是否小于或大于123。
* 然后你可以用它来执行进一步的操作。
* 特别是，将这样的“布尔值”相乘或相加，分别得到 AND 和 OR 逻辑运算符。

通过有条件设置 `numParticles` 可以实现的一些明显用例：
* 你可以通过检查剩余的TTL是否足够低，使CEG轨迹在正常弹射时间之前提前失效。
* 在通过伤害操作符用于多种类似武器的CEG中，对于足够大的爆炸，可以启用额外的粒子类型。
* 比较运算和布尔逻辑带来了许多可能性，使其成为或许最强大的运算符。

### 乘方（`p`）和乘方缓冲区（`q`）

`p` 操作符将当前值提升到操作数的幂次。`q` 操作符类似，但取幂来自指定的缓冲区。主要用途可能是计算 x² 或 √x，以使体积效果更偏向于中心集中或偏心分布。示例：
 * `3 p4` 是81，因为是3⁴。
 * `4 y7 3 q7` 也是81（并以4的值离开第7个缓冲区槽位）。
